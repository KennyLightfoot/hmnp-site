## Contributing to Documentation

When adding new documentation:

1. Use clear, descriptive filenames in UPPERCASE_WITH_UNDERSCORES.md format
2. Include a title and brief description at the top of each document
3. Organize content with appropriate headings (H1, H2, H3)
4. Add your new document to this index if it's a major guide
5. Keep content updated as the system evolves

## Documentation Standards

- Use Markdown for all documentation
- Include code examples when relevant
- Document both "how" and "why" when explaining features
- Include troubleshooting sections for complex features
- Update dates when making significant changes
